## Top findings (TL;DR)

### What hurts most (cross-competitor)

1. **AI makes voices sound wrong** — robotic, metallic, muffled, digitized, "Elmer Fudd-like," listener fatigue on long episodes
2. **Room sound is the root problem** — echo, reverb, "recorded in a cave/bathroom," untreated home offices
3. **Tools get worse after updates** — no rollback, quality regressions, users switching between Adobe ↔ Descript
4. **Subscription fatigue** — "scam," credits expiring, daily limits, paying for a full suite when they only need cleanup
5. **Cloud = privacy anxiety** — uploading interviews, sermons, confidential recordings to third-party servers
6. **Workflow fragmentation** — Adobe for noise → Cleanvoice for fillers → Audacity for levels → iZotope for disasters
7. **No control knob** — binary output, aggressive defaults, free tiers locked out of intensity sliders
8. **Real-time tools drain hardware** — NVIDIA Broadcast GPU idle usage, Krisp choppy/robotic under load

### Luma positioning hooks (from customer language)

| They say… | Luma can answer with… |
|-----------|----------------------|
| "Sounds robotic / metallic / over-processed" | Local processing tuned for **voice character preservation** |
| "I don't want to upload my audio" | **On-device desktop app** — files never leave the hard drive |
| "Another subscription for one feature" | **One-time purchase** (if that's the pricing direction) |
| "I just need to fix my recording, not learn RX" | **Post-recording cleanup** without pro audio engineering |
| "Studio Sound got worse after the update" | Stable local tool without surprise cloud model changes |
| "It works on terrible audio but ruins good audio" | Selective enhancement / intensity control narrative |
